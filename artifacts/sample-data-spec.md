# Sample Data Spec

## Purpose
Define the fake records required to demo the workflow safely.

## Records
- 5 new leads with complete data and mixed quality.
- 2 duplicate submissions matching an earlier email/source pair.
- 2 incomplete leads missing either email or clear intent.
- 1 ambiguous luxury lead to test low-confidence handling.

## Required Fields
- full name
- email
- phone
- source channel
- property type
- budget range
- timeline
- free-text message

## Fake Data Rules
- Use non-real emails under example domains.
- Use fictional addresses and neighborhoods.
- Avoid real phone numbers and any client-identifying information.
